Use these layers as a reference to back the existing HTML surfaces with a hardened service mesh. Each component is defensive-only: no unsolicited outbound traffic, and every call carries compliance locks plus audit trails.

Edge Gateway mTLS + JWT, rate limits, geo/IP allowlists
  • Terminates TLS 1.3
  • Injects request IDs & traceparent
  • Routes to internal mesh only
Policy Orchestrator Wraps requests with Compliance Core locks
  • Policy bundles signed + versioned
  • Denies non-compliant payloads pre-ingest
  • Feeds violations to Audit Core
Signal Bus Event backbone with replay-safe streams
  • Append-only topics for AML, NDSP
  • Hash-linked ledgers for investigations
  • Fan-out to analytics / cold storage
Data Plane Encrypted stores and tokenized secrets
  • Row-level encryption with quantum-safe KDF
  • Vaulted credentials; short-lived leases
  • Immutable audit evidence store
Deployment hint: Isolate Edge Gateway in a DMZ, run Policy Orchestrator and Signal Bus on a private network segment, and keep the Data Plane behind a dedicated bastion with hardware-backed keys.

Request Flow & UX Touchpoints

Traceable UX Read-only defaults Composable widgets
1) UI Client

HTML modules call /api/compliance/locks, /api/audit/run, /api/aml/score, and /api/ndsp/telemetry through the Edge Gateway.

2) Gateway Guards

mTLS + JWT validation, rate limits, geo fences. Attaches X-Request-ID and traceparent to every call.

3) Policy Orchestrator

Loads the Compliance Core bundle, enforces anti-crime and non-intrusion locks, then forwards to the right microservice if checks pass.

4) Domain Services

Dedicated pods: audit-service, aml-service, telemetry-service. Each emits evidence to the Audit Core ledger and the Signal Bus.

UX guidance: Keep UI states tied to server signals: show gateway verdicts, policy enforcement notes, and evidence IDs inline. Favor read-only previews first, with explicit opt-in buttons for mutating actions.

API Skeleton

Bootstrap a defensive API surface with predictable routes and JSON schemas. Replace handlers with your runtime of choice (Node, Go, Python) but keep the contract stable.

POST /api/compliance/locks -> validate payload, return {status, locks, evidenceId} POST /api/audit/run -> run audit, stream hash chain to ledger, return signed result POST /api/aml/score -> score entity/transaction, return verdict + reasons + evidenceId GET /api/ndsp/telemetry -> read-only drift + health snapshots, never mutating
Evidence pattern: Every POST writes an immutable record to the Audit Core evidence store via the Signal Bus. Use request IDs and hash-linked entries so UI screens can deep-link to signed receipts.
